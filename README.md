# History-project

# Описание проекта

Интерактивная карта взаимодействия исторических событий и личностей. 

Карта позволит динамически добавлять новые сущности, которые в контексте проекта могут представлять собой различных исторических личностей, события, предметы искусства и тому подобное.

Между любыми двумя сущностями можно добавить связь (описание того, как эти две сущности связанны между собой)

# Проблематика

Полученный проект позволит визуализировать и моделировать исторические связи, что может быть полезно при изучении истории

# Задание

- [ ] Разработать модель интерактивной карты, которая позволяет добавлять сущности и связи между ними.
- [ ] Реализовать представление полученной модели в виде сайта
- [ ] Заполнить, получившуюся карту сущностями из истории России 20 века, чтобы продемонстрировать работу системы

# Функциональные требования

## Сущность

Представляет собой объект с какой-то информацией. Должен иметь следующие атрибуты:
- Название
- Краткое описание
- Статья (опционально)
- Изображение (опционально) 
- Тег (опционально)

Должна быть возможность перейти к статье, ассоциированной с сущностью, при её наличии

Должна быть возможность добавлять связи с другими сущностями и переходить по ним

Должна быть возможность указать тег сущности

## Связь

- Связь связывает две сущности между собой. 
- Имеет описание. 

### Виды связей

Связи могут быть двух видов

#### Двухсторонняя связь

- Связывает две равноправные сущности между собой

#### Односторонняя связь

- Связывает две сущности с разными правами (например художник и его картина). 
- Односторонние связи могут не иметь описание

Права определяются пользователем, добавляющим новые связи

## Статья

Более подробное описание сущности. Имеет текст и изображения. В конце статье должен быть список всех ассоциируемых сущностей

## Каталог

Каталог содержит в себе список всех сущностей, автоматически группирует их по тегам и позволяет переходить по сущностям.

Например, там могут быть следующие группы: "Личность", "Картина", "Событие". Пользователь может нажать на "Личность" и ему откроется список всех сущностей с соответствующим тегом. 

Если сущность имеет односторонние связи (например, добавляем художнику картины), то она сама становится подгруппой в каталоге и при нажатии на нее развернется список ассоциируемых "подсущностей"

## Фильтр

Фильтр позволяет фильтровать сущностей по тегам. На карте отображаются только те сущности, которые удовлетворяют требуемым критериям фильтрации

# Нефункциональные требования

- Приложение должно иметь луковую архитектуру с богатой доменной моделью
- Приложение должно следовать основным принципам разработки ПО, GRASP, SOLID, паттерны при необходимости
- Добавление новой сущности должно быть простым
- Добавление атрибутов сущности должно быть простым
- Добавление новых связей между сущностями должно быть простым
- Приложение не должно терять производительности при добавлении большого количества сущностей и связей, можно использовать кеширование, либо выгружать из памяти неиспользуемые сущности
- Карта не должна выглядеть "перегружено" при добавлении большого количества сущностей
- Должна быть визуальная зависимость размера отображаемой сущности, от количества связей
- Односторонние связи должны весить меньше, чем двухсторонние
- Визуально карта должна выглядеть приятно, не перегружено, должно быть продумано как сделать организацию всех карточек удобно и приятно для конечного пользователь
